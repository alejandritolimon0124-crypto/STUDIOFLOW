# Studio Flow Development Guidelines

## Execution Mode

Continuamos en:
- STRICT EXECUTION MODE
- STABLE FOUNDATION MODE

## Reglas Obligatorias

1. NO modificar layouts funcionales existentes.
2. NO modificar navegación ya funcional.
3. NO modificar responsive aprobado.
4. NO rehacer dashboards.
5. NO reorganizar arquitectura.
6. NO reinterpretar UX aprobada.
7. NO crear demos nuevas.
8. NO renderizar paneles duplicados.
9. NO cambiar estilos globales aprobados.
10. NO modificar componentes existentes salvo instrucción explícita.
11. NO expandir funcionalidades automáticamente.
12. NO tomar iniciativa creativa.
13. NO modificar App.jsx salvo instrucción explícita.
14. NO alterar rutas funcionales existentes.
15. NO modificar lógica auth ya funcional.
16. NO tocar bottom navigation funcional.
17. NO tocar topbars funcionales.
18. NO cambiar estructura mobile-first aprobada.

## Modo de Trabajo

- Cambios quirúrgicos.
- Modificar únicamente archivos solicitados.
- Mantener intacto todo lo ya funcional.
- Priorizar estabilidad y continuidad.

NO tocarlo.
